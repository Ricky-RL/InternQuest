What should the Profile page do in the MVP?

No backend — all state is local/mock. Pick the scope that fits the MVP.

A

Minimal — Static Demo

A pre-filled mock profile showing name, skills, university, and a placeholder resume. Not editable. Just demonstrates what the feature will look like.

B

Interactive Form (Recommended)

User can fill in name, university, skills, experience. Saved to localStorage so it persists across refreshes. Resume upload is UI-only (no actual processing). Shows a mock "match score" on job cards based on selected skills.

C

Full Flow — Resume + Mock AI Score

Everything in B, plus a visible "AI compatibility score" badge on job cards that adjusts based on the skills the user enters. Purely cosmetic calculation, but demos the core product vision.
